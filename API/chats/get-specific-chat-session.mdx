title: "Get Specific Chat Session"
description: "Fetches a particular chat session by its session_id, along with all messages inside it."
---

**GET** `/chats/sessions/{session_id}`

This endpoint allows the authenticated user to access a **specific chat session** using its unique `session_id`.  
It returns:

- The **session metadata**  
- The **messages** belonging to that session  

A valid authentication token is required.

# üß© Top-Level Response Structure

| Field    | Datatype          | Required | Description |
|----------|--------------------|----------|-------------|
| session  | object            | Yes      | Metadata describing the chat session |
| messages | Array of objects  | Yes      | List of messages belonging to the session |

---

# üß© Session Object Fields

| Field         | Datatype | Required | Description |
|---------------|----------|----------|-------------|
| session_uuid  | string   | Yes      | Unique ID of the chat session |
| title         | string   | Yes      | Title of the chat session |
| created_at    | string   | Yes      | Timestamp when the session was created |
| updated_at    | string   | Yes      | Timestamp when the session was last updated |
| is_pinned     | boolean  | Yes      | Whether the session is pinned |
| persona_name  | string   | No       | Persona associated with the chat |
| is_pending    | boolean  | Yes      | Whether the session is pending |

---

# üß© Messages Array

Each element of the `messages` array represents a single message.

> Because your example shows an empty list, message field structure will be documented in endpoint 6 (`create-chat-query`) when we receive that one.

# ‚ö†Ô∏è Possible Errors

| Status | Description |
|--------|-------------|
| 400 Bad Request | Invalid session_id format. |
| 401 Unauthorized | Token missing or invalid. |
| 404 Not Found | Session does not exist or does not belong to the user. |
| 500 Internal Server Error | Unexpected backend issue. |

---

# üìò Notes

- The session must belong to the authenticated user; otherwise the backend returns **404**.  
- Empty `messages` array indicates that the user has not added messages yet.  
- Useful for opening a chat session in the UI and loading its content.
