---
title: "Create Note Item (Audio Upload)"
description: "Create a new note item under an existing note by uploading an audio file. Supports optional transcription."
---

**POST** `/notes/{note_uuid}/items/`

Creates a **new note item** inside an existing note using an **audio file** (`.m4a`, `.wav`, etc.).  
Supports optional **transcription** by setting `transcribe=true`.  
The server generates the note item using your provided `item_uuid` and attaches the uploaded audio.

---

## üîê Required Headers

| Key           | Value Format           | Required | Description |
|--------------|-------------------------|----------|-------------|
| Authorization | `Token <token>`        | Yes      | User authentication token |
| Content-Type  | `multipart/form-data`  | Yes      | Required for uploading audio |

---

## üì§ Form-Data Fields

| Field      | Datatype | Required | Description |
|------------|----------|----------|-------------|
| file       | file (audio) | Yes | The audio file to upload |
| item_uuid  | string   | Yes | UUID for the new note item |
| transcribe | string   | No  | `"true"` or `"false"` ‚Äî Enables transcription |
| content    | string   | No  | Optional text content (can be empty) |

---

## üìù Example Form-Data (JSON representation)

```json
[
  { "key": "file", "type": "file", "value": ["sample-3.m4a"] },
  { "key": "item_uuid", "type": "text", "value": "78b7e422-81b5-4286-a774-76f3f58fc6d4" },
  { "key": "transcribe", "type": "text", "value": "false" },
  { "key": "content", "type": "text", "value": "" }
]
```

---

## üìÆ Example cURL

```bash
curl -X POST "https://your-api.com/notes/{note_uuid}/items/" \
  -H "Authorization: Token <token>" \
  -F "file=@sample-3.m4a" \
  -F "item_uuid=78b7e422-81b5-4286-a774-76f3f58fc6d4" \
  -F "transcribe=false" \
  -F "content="
```
