**GET** `/auth/magic-link/verify`

This endpoint verifies the **magic link token** sent to the user's email.  
When the user clicks the magic link, the token attached to the URL is used to:

- Validate the user  
- Log them into the application  
- Return a fresh authentication token  
- Return the complete user object  

This is the final step of magic link login.

---

## üß© Magic Link Format

The token is passed **as a query parameter** in the URL:

```

```

# üìò Notes

- Tokens in magic links are **single-use** and may expire.  
- On successful verification, the returned `"token"` should be stored for authenticated requests.  
- Magic link login eliminates the need for passwords.
