**POST** `/auth/social-login/`

This endpoint authenticates users using **Apple Sign-In**.  
The client must send the Apple-generated **id_token** along with `"provider": "apple"` to verify the user.

After validation, the server will either:

- Create a new user (if first login), or  
- Log in the existing user  

Then the backend returns:

- An application-level **session token**
- The **full user profile**

---

## üìù Request Body Parameters

| Field       | Datatype | Required | Description |
|-------------|----------|----------|-------------|
| provider    | string   | Yes      | Must be `"apple"` to indicate Apple social login. |
| access_token | string  | No       | Optional Apple access token (some configurations send it). |
| id_token    | string   | Yes      | The JWT ID token returned by Apple Sign-In, containing user identity information. |

# üìò Notes

- Apple sends user details **only on the first login**.  
- The backend must validate the `id_token` signature against Apple‚Äôs public keys.  
- The returned `"token"` is your backend‚Äôs session token, not Apple‚Äôs token.
