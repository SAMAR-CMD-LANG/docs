**POST** `/chat/sessions/`

This endpoint allows the authenticated user to **create a new chat session**.  
A valid token must be provided in the `Authorization` header.

The request body contains:

- A **title** for the chat session (auto-generated or user-defined)  
- A boolean **is_pinned** flag indicating whether the chat is pinned

# ğŸ§© Chat Session Fields Returned by API

Below are the fields returned after creating a session.

| Field          | Datatype | Nullable | Description |
|----------------|----------|----------|-------------|
| session_uuid   | string   | No       | Unique identifier of the chat session. |
| title          | string   | No       | Title of the created chat. |
| created_at     | string   | No       | Timestamp when the session was created. |
| updated_at     | string   | No       | Timestamp when the session was last updated. |
| is_pinned      | boolean  | No       | Indicates whether the session is pinned. |
| persona_name   | string   | Yes      | Persona associated with the session (if any). |
| is_pending     | boolean  | No       | Whether session creation is pending. |
