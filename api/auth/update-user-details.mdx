**PATCH** `/auth/user`

This endpoint updates the authenticated user‚Äôs profile information.  
It requires a valid authentication token and **multipart/form-data** since profile picture uploads involve file input.

### Key Notes:
- The request **must** be sent as `multipart/form-data`.
- Only fields included in the form will be updated.
- `profilepic` must be sent as a real file (image) ‚Äî **not base64**.
- A valid `Authorization: Token <token>` header is required.

---

## üîê Required Headers

| Header        | Datatype | Required | Description |
|--------------|----------|----------|-------------|
| Authorization | string   | Yes      | Must include: `Token <user_token>`. |
| Content-Type  | string   | Yes      | Must be `multipart/form-data` because the request contains file upload. |

---

## üìù Request Body Parameters (multipart/form-data)

| Field       | Datatype        | Required | Description |
|-------------|-----------------|----------|-------------|
| first_name  | string          | No       | Updated first name of the user. |
| last_name   | string          | No       | Updated last name of the user. |
| profilepic  | file (image)    | No       | Image file (jpg, png, jpeg). |

---

## üìå Example Headers

```bash
Authorization: Token 6a0110b06109fbf2e3a7046e4ace26a7c7ed3cda
Content-Type: multipart/form-data
```

---

## üìÆ Example cURL (Multipart Form)

```bash
curl -X PATCH "https://your-api.com/auth/user" \
  -H "Authorization: Token 6a0110b06109fbf2e3a7046e4ace26a7c7ed3cda" \
  -F "first_name=samarjeeth" \
  -F "last_name=RS" \
  -F "profilepic=@/path/to/image.jpg"
```
