**GET** `/notes/`

This endpoint retrieves a **paginated list of all notes** created by the authenticated user.  
Each note returned includes summary information such as the title, category, pinned status, and a preview of the first note item's content.

Pagination is handled using `count`, `next`, `previous`, and `results`.

# üß© Response Body Fields (Top Level)

| Field    | Datatype       | Required | Description |
|----------|----------------|----------|-------------|
| count    | number         | Yes      | Total number of notes |
| next     | string / null  | Yes      | URL for next page (or `null`) |
| previous | string / null  | Yes      | URL for previous page (or `null`) |
| results  | array          | Yes      | List of note objects |

---

# üß© Note Object (results[]) Fields

| Field             | Datatype | Required | Description |
|-------------------|----------|----------|-------------|
| note_uuid         | string   | Yes      | Unique identifier of the note |
| title             | string   | Yes      | Title of the note |
| category          | number   | Yes      | Category ID |
| labels            | array    | Yes      | Array of label IDs |
| is_pinned         | boolean  | Yes      | Whether the note is pinned |
| first_item_content | string  | Yes      | Preview of the first note item |
| created_at        | string   | Yes      | Timestamp when note was created |
| updated_at        | string   | Yes      | Timestamp when note was last updated |

# üìò Notes

- Pagination keys (`next`, `previous`) allow lazy-loading large lists.  
- `first_item_content` is useful to preview a note without fetching items.  
- To get full note content, you must call **Get Single Note**.
