# POST `/notes/{note_uuid}/attachments/`

Attach a file (audio, image, or document) to an existing note.  
The uploaded file is stored, and the note content is automatically updated with an `<attachment-embed>` block.

---

## üîê Required Headers

| Key | Value Format | Required | Description |
|------|--------------|----------|-------------|
| Authorization | Token `<token>` | Yes | Authenticates the user |
| Content-Type | multipart/form-data | Yes | Required for file uploads |

---

## üì§ Form Data Fields

| Field | Type  | Required | Description |
|--------|--------|----------|-------------|
| file   | File  | Yes      | The attachment file (audio, image, document) |

---

## üì• Response ‚Äî 200 OK

### Response Fields

| Field            | Type         | Description |
|------------------|--------------|-------------|
| note_uuid        | UUID         | Unique identifier of the note |
| title            | String       | Title of the note |
| category         | Integer      | Category assigned to the note |
| labels           | Array        | Labels applied to the note |
| is_pinned        | Boolean      | Whether the note is pinned |
| is_public        | Boolean      | Whether the note is public |
| content          | HTML String  | Content including embedded attachment block |
| attachments      | Array        | List of attachment metadata |
| ai_content_items | Array        | AI-generated content items |
| created_at       | ISO datetime | Timestamp of note creation |
| updated_at       | ISO datetime | Timestamp after attachment upload |

---

### Attachment Object Fields

| Field      | Type         | Description |
|------------|--------------|-------------|
| id         | Integer      | Internal attachment ID |
| url        | String (URL) | Direct file link |
| type       | String       | Type of attachment (`audio`, `image`, `document`) |
| size       | Integer      | File size in bytes |
| name       | String       | Original filename |
| created_at | ISO datetime | Upload timestamp |
