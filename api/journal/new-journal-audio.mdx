---
title: "Create New Journal (Audio)"
description: "Creates a journal entry with an audio attachment, with optional automatic transcription."
---

**POST** `/journals/{date}/`

This endpoint creates a **journal entry with an audio attachment** for a specific date.  
It accepts **multipart/form-data** and supports optional transcription of the audio file.

Only the authenticated user can create audio journals.

If `"transcribe": "true"` is passed, the backend will automatically transcribe the audio and store the generated text in the `content` field.

---

## üîê Required Headers

| Key           | Value Format         | Required | Description |
|---------------|----------------------|----------|-------------|
| Authorization | `Token <token>`      | Yes      | User authentication token |
| Content-Type  | `multipart/form-data` | Yes     | Required for file upload |

---

## üìù Form-Data Fields

| Field        | Datatype | Required | Description |
|--------------|----------|----------|-------------|
| transcribe   | string (`"yes"` or `"no"`) | No | If `"true"`, backend will generate transcription |
| audio_file   | file     | Yes      | Audio file to upload (`.m4a`, `.mp3`, `.wav`, etc.) |
| content      | string   | Yes      | Optional extra text content to save with audio |

---

## üìå Example Form-Data

| Key         | Value                     |
|-------------|----------------------------|
| transcribe  | true                       |
| audio_file  | `<uploaded_audio_file>`    |
| content     | hello                      |

---

## üìÆ Example cURL

```bash
curl -X POST "https://your-api.com/journals/2025-03-11/" \
  -H "Authorization: Token 6a0110..." \
  -F "transcribe=true" \
  -F "audio_file=@/path/to/file.m4a" \
  -F "content=hello"
```

# üß© Response Object Fields

| Field        | Datatype | Required | Description |
|--------------|----------|----------|-------------|
| journal_uuid | string   | Yes      | Unique identifier for the journal entry |
| content      | string   | Yes      | Saved content (text or transcription result) |
| date         | string   | Yes      | Journal date (YYYY-MM-DD) |
| attachment   | array    | Yes      | Audio attachment info |
| created_at   | string   | Yes      | Timestamp of journal creation |
| updated_at   | string   | Yes      | Timestamp of last update |
